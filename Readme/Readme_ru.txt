Large Folder Finder
====================
Инструмент для быстрого анализа и составления списка иерархий папок.
Полезен для анализа папок с использованием условий размера и фильтров (подстановочные знаки, регулярные выражения).
Помогает определить причину проблем в больших данных, используемых несколькими пользователями, таких как NAS.


■ Как использовать
--------------------
  1. Выберите папку, которую хотите исследовать.
  2. Нажмите кнопку "▶" (Сканировать), чтобы начать поиск.
  3. Результаты отображаются в формате, похожем на Проводник Windows.
  4. Укажите условия отображения: минимальный размер для извлечения, фильтр, сортировка, свернуть папки.
  5. Нажмите кнопку копирования в правом верхнем углу, чтобы скопировать результаты отображения в буфер обмена.
  6. Нажмите кнопку "+" справа от вкладки, чтобы начать новое сканирование, сохраняя историю.
    История сохраняется даже после закрытия приложения.
※ Запуск приложения с правами администратора позволяет анализировать папки с правами администратора на диске C.
※ Вы можете переключить язык или изменить макет в Меню/Вид.
※ Вы можете изменить настройки в Меню/Вид/Открыть расширенные настройки(S). Подробности описаны ниже.


■ О функциях отображения
-------------------
1. Сортировка
  Щелкните каждую метку (Имя, Размер, Дата изменения, Тип), чтобы отсортировать порядок отображения.
  Щелкните еще раз, чтобы переключиться между возрастанием/убыванием.
2. Показать файлы
  Отметьте это, чтобы отображать также файлы.
3. Минимальный размер
  Укажите минимальный размер папок или файлов для отображения. Будут отображаться элементы, равные или превышающие установленное значение.
  Введите 0, если хотите отобразить все.
  Единицы можно выбрать от Byte до TB.
4. Фильтр
Подстановочные знаки: Такое же поведение, как в Проводнике Windows.
  * Позволяет соответствовать любой строке. Пример) *.txt Все txt-файлы с любым именем. Пример 2) *данные* Все файлы с "данные" в имени.
  ? Позволяет соответствовать любому одному символу. Пример) 202?год → 2020год~2029год и т.д. (также соответствует не цифрам)
  ~ Поместите перед (* или ?), чтобы искать эти символы сами по себе. Пример) ~?.txt → Ищет ?.txt
Регулярное выражение: Расширенная функция фильтра (используется инженерами и т.д.)
  Может делать то, что подстановочные знаки не могут. Соответствовать только цифрам, строчным буквам, заглавным буквам, извлекать только несоответствующие элементы и т.д.
  Это сложно, поэтому поищите "как использовать регулярные выражения" отдельно.
  Также доступны инструменты проверки регулярных выражений для проверки правильности работы вашего поиска.
5. Пробел/Табуляция
  Укажите, заполнять ли пространство между именем и размером пробелами или табуляциями при нажатии кнопки копирования.


■ Когда не работает должным образом
------------------------
※ Вы можете проверить поведение приложения в Меню/Вид/Журналы.
※ Если приложение ведет себя странно, удаление данных в следующей папке может сбросить кэш и восстановить функциональность.
    %LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder


■ О расширенных настройках (Config.txt)
--------------------
Редактируя "Config.txt" в каталоге выполнения, можно настроить более детальное поведение.
Нажмите кнопку "⚙" в пользовательском интерфейсе, чтобы немедленно открыть его в текстовом редакторе, таком как Блокнот.
Конфигурация должна следовать формату YAML. Если вы хотите добавить свои собственные комментарии, поставьте перед ними #.

    ▽ Настраиваемые элементы: (По умолчанию)
    UseParallelScan: true
        Тип: bool (true/false)
        Описание: Включить параллельную обработку
        Ожидаемое значение (true): Эффективно для NAS (сетевое хранилище) и т.д. Локальные SSD быстрые, поэтому накладные расходы на параллелизацию могут быть больше.

    SkipFolderCount: false
        Тип: bool (true/false)
        Описание: Пропускать ли предварительный подсчет для отображения прогресса и немедленно начинать сканирование
        Если true, процент прогресса не может быть отображен, потому что общее число неизвестно.

    MaxDepthForCount: 3
        Тип: int (натуральное число)
        Описание: Максимальная глубина иерархии для предварительного подсчета папок для определения процента прогресса
        Большая указанная иерархия может занять больше времени. Вместо этого точность прогресса улучшается.
        Ожидаемое значение (3): NAS: 3~6, Внутренний ПК: 7~

    UsePhysicalSize: true
        Тип: bool (true/false)
        Описание: Рассчитывать ли "выделенный размер на диске" с учетом размера кластера
        Ожидаемое значение (true): Обычно рекомендуется оставить true. Результаты будут ближе к отображению свойств Windows. Если false, рассчитывается по размеру файла.
        Перед настройкой этого мы рекомендуем запустить от имени администратора. Системные файлы будут включены в расчеты для повышения точности.

    OldDataThresholdDays: 30
        Тип: int (Неотрицательное целое число)
        Описание: Выделяет вкладку желтым цветом, чтобы указать на устаревшие данные сканирования, если прошло указанное количество дней.
        Ожидаемое значение: На усмотрение пользователя.

■ Как добавить языковые файлы
--------------------
Этот инструмент поддерживает несколько языков, и вы можете добавлять новые.
1. Откройте папку "Languages" в той же иерархии, что и исполняемый файл приложения (.exe).
2. Скопируйте существующий файл, такой как "en.yaml", и переименуйте его в код культуры языка, который вы хотите добавить (например, "fr.yaml" для французского).
   * Список кодов культур (например: ja-JP / ja) см. по адресу:
   https://learn.microsoft.com/ru-ru/windows-hardware/manufacture/desktop/available-language-packs-for-windows?view=windows-11
3. Отредактируйте текст в файле YAML (сохраните в формате UTF-8).
4. Перезапустите приложение, и новый язык появится в меню "Language".
※ При необходимости создайте и добавьте Readme_<language_code>.txt, ссылаясь на другие файлы.


■ Полная деинсталляция (Удаление настроек и журналов)
--------------------
Чтобы полностью удалить настройки и журналы выполнения этого инструмента, вручную удалите следующую папку:
%LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder
(Вы можете открыть ее напрямую, вставив указанный выше путь в адресную строку Проводника)
