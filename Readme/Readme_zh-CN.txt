Large Folder Finder
====================
快速分析和列出文件夹层次结构的工具。
使用大小条件和筛选器(通配符、正则表达式)进行文件夹分析很有用。
有助于识别NAS等多人使用的大数据中的问题原因。


■ 使用方法
--------------------
  1. 选择要调查的文件夹。
  2. 按"▶"(扫描)按钮开始搜索。
  3. 结果以类似Windows资源管理器的格式显示。
  4. 指定显示条件:要提取的最小大小、筛选器、排序、折叠文件夹。
  5. 按右上角的复制按钮将显示结果复制到剪贴板。
  6. 按选项卡右侧的"+"按钮,在保留历史记录的同时开始新扫描。
    即使关闭应用程序,历史记录也会保留。
※ 以管理员权限运行应用程序可以分析C盘内的管理员权限文件夹。
※ 可以从菜单/查看切换语言或更改布局。
※ 可以从菜单/查看/打开高级设置(S)更改设置。详细信息如下所述。


■ 关于显示功能
-------------------
1. 排序
  单击每个标签(名称、大小、修改日期、类型)对显示顺序进行排序。
  再次单击可在升序/降序之间切换。
2. 显示文件
  选中此项也会显示文件。
3. 最小大小
  指定要显示的文件夹或文件的最小大小。将显示等于或大于设置值的项目。
  如果要显示全部,请输入0。
  单位可从Byte到TB选择。
4. 筛选器
通配符:与Windows资源管理器相同的行为。
  * 允许匹配任意字符串。例) *.txt 所有名称的txt文件。例2) *数据* 名称中包含"数据"的所有文件。
  ? 允许匹配任意单个字符。例) 202?年 → 2020年~2029年等。(也匹配非数字)
  ~ 要搜索(*或?)本身时,放在其前面。例) ~?.txt → 搜索?.txt
正则表达式:高级筛选功能(工程师等使用)
  可以做通配符无法做到的事情。仅匹配数字、小写字母、大写字母,仅提取不匹配的项目等。
  由于复杂,请另外搜索"如何使用正则表达式"。
  还有正则表达式检查工具,可用于验证搜索是否正确工作。
5. 空格/制表符
  按复制按钮时,指定用空格还是制表符填充名称和大小之间的空间。


■ 无法正常工作时
------------------------
※ 可以从菜单/查看/日志检查应用程序的行为。
※ 如果应用程序行为异常,删除以下文件夹中的数据可能会重置缓存并恢复功能。
    %LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder


■ 关于高级设置 (Config.txt)
--------------------
通过编辑执行目录中的"Config.txt",可以进行更详细的行为设置。
单击UI上的"⚙"按钮可立即使用记事本等文本编辑器打开它。
设置必须遵循YAML格式。如果要添加自己的注释,请在前面加上#。

    ▽ 可配置项目: (默认值)
    UseParallelScan: true
        类型: bool (true/false)
        说明: 启用并行处理
        预期值 (true): 对NAS(网络存储)等有效。本地SSD速度快,并行化开销可能更大。

    SkipFolderCount: false
        类型: bool (true/false)
        说明: 是否跳过进度显示的预计数并立即开始扫描
        如果为true,则无法显示进度百分比,因为总数未知。

    MaxDepthForCount: 3
        类型: int (自然数)
        说明: 为确定进度百分比而预先计数文件夹的最大层次深度
        指定的层次越大可能需要更多时间。但进度准确性会提高。
        预期值 (3): NAS: 3~6, PC内部: 7~

    UsePhysicalSize: true
        类型: bool (true/false)
        说明: 是否考虑簇大小计算"磁盘上的分配大小"
        预期值 (true): 通常建议保持true。结果将更接近Windows属性显示。如果为false,则按文件大小计算。
        在调整此设置之前,建议以管理员身份运行。系统文件也将包含在计算中以提高准确性。

    OldDataThresholdDays: 30
        类型：int (非负整数)
        说明：如果超过指定的天数，会将标签页高亮显示为黄色，以表示扫描数据已过期。
        预期值：用户偏好。

■ 如何添加语言文件
--------------------
此工具支持多种语言,您可以添加新语言。
1. 打开与应用程序可执行文件(.exe)相同层次结构中的"Languages"文件夹。
2. 复制现有的"en.yaml"等文件,并将文件名更改为要添加的语言的区域性代码(例如,法语为"fr.yaml")。
   * 有关区域性代码列表(例如: ja-JP / ja),请参阅以下内容:
   https://learn.microsoft.com/zh-cn/windows-hardware/manufacture/desktop/available-language-packs-for-windows?view=windows-11
3. 编辑YAML文件中的文本(以UTF-8格式保存)。
4. 重新启动应用程序,新语言将显示在"Language"菜单中。
※ 如有必要,请参考其他文件创建并添加Readme_<language_code>.txt。


■ 完全卸载 (删除设置和日志)
--------------------
要完全删除此工具的设置和执行日志,请手动删除以下文件夹:
%LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder
(您可以通过将上述路径粘贴到资源管理器地址栏中直接打开它)
