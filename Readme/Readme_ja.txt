Large Folder Finder
====================
フォルダ階層を高速に解析して一覧表示するツールです。
サイズ条件やフィルタ(ワイルドカード、正規表現)を利用して、フォルダ解析に役立ちます。
NASなど複数人で利用する大きなデータで、問題の原因を特定するのに役立ちます。


■ 使い方
--------------------
  1. 調査したいフォルダを選択します。
  2. 「▶」(スキャン)ボタンを押すと探索が始まります。
  3. 結果はエクスプローラーの見た目に近い形式で表示されます。
  4. 表示条件を指定します。抽出したい最小サイズ。フィルタ。ソート。フォルダを閉じる。
  5. 右上のコピーボタンを押すと、表示結果をクリップボードにコピーできます。
  6. タブの右にある「+」ボタンを押すことで、履歴を残したまま新たにスキャンを開始することができます。
    履歴はアプリケーションを終了しても保持されます。
※ アプリを管理者権限で実行することで Cドライブ内 の管理者権限フォルダも解析できるようになります。
※ メニュー/表示 から言語を切り替えたり、レイアウトの変更が可能です。
※ メニュー/表示/高度な設定を開く(S) から設定を変更できます。詳しくは下記に記載。


■ 表示機能について
-------------------
1. ソート
  各ラベル(名前、サイズ、更新日時、種類)をクリックすると表示順を並べ替えます。
  もう一度押すと昇順/降順を切り替えます。
2. ファイルも表示
  チェックを入れることで、ファイルも表示するようにします。
3. 最小サイズ
  表示する最小のサイズのフォルダやファイルを指定します。設定値以上のものが表示されるようになります。
  全て表示したい場合は、0を入力してください。
  単位は Byte ~ TB まで選択できます。
4. フィルタ
ワイルドカード:Windowsエクスプローラーと同じ挙動です。
  * 任意の文字列と一致することを許容します。 例) *.txt 全ての名前のtxtファイル。 例2) *データ* 名前に"データ"が含まれる全てのファイル。
  ? 任意の1文字と一致することを許容します。例) 202?年 → 2020年～2029年など。(数字以外も一致するけど) 
  ~ (*や?)自体を検索したい場合に、その前に置く。　例) ~?.txt → ?.txtが検索される
正規表現:高度なフィルタ機能(エンジニアなどが利用します)
  ワイルドカードではできないこともできます。数字・アルファベット小文字・大文字だけなどの一致、一致しないものだけを抽出など。
  複雑なため別途「正規表現の使い方」などで検索してみてください。
  正規表現チェックツールなどもあり、正しく検索できているか活用してみてください。
5. 空白/タブ
  コピーボタンを押した際に、名前とサイズの間を埋める文字列を空白かタブか指定します。


■ 上手く動作しないときは
------------------------
※ メニュー/表示/ログ からアプリの挙動を確認することができます。
※ アプリの挙動がおかしいときは、以下のフォルダ内のデータを削除すると、キャッシュがリセットされ動作することがあるかもしれません。
    %LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder


■ 高度な設定について (Config.txt)
--------------------
実行ディレクトリにある「Config.txt」を編集することで、より詳細な動作設定が可能です。
UI上の「⚙」ボタンをクリックすることで、メモ帳などのテキストエディタで即座に開くことができます。
設定方法はYaml形式に則り行ってください。独自でコメントを記載したい場合は、#を先頭につけてください。

    ▽設定可能な項目：(既定値)
    UseParallelScan: true
        型：bool (true/false)
        説明：並列処理の有効化
        想定値 (true): NAS（ネットワークストレージ）で効果を発揮します。ローカルSSDは高速なため、並列化のオーバーヘッドの方が大きい場合があります。

    SkipFolderCount: false
        型：bool (true/false)
        説明：進捗率表示用の事前カウントをスキップして即座にスキャンを開始するか
        true の場合は総数を把握できないため、進捗率は表示されません。

    MaxDepthForCount: 3
        型：int (自然数)
        説明：進捗率を把握するために、事前にフォルダ総数カウントを行う最大階層
        指定した階層が大きいと時間がかかる可能性があります。代わりに進捗率の精度が上がります。
        想定値 (3): NAS: 3~6, PC内 : 7~

    UsePhysicalSize: true
        型：bool (true/false)
        説明：クラスタサイズを考慮して「ディスク上の占有サイズ」を計算するかどうか
        想定値 (true): 通常は true 推奨。Windowsのプロパティ表示に近い値になります。falseだとファイルのサイズで計算されます。
        この設定を見直す前に、管理者権限で実行することをお勧めします。システムファイルも含めて計算され正確になります。
    
    OldDataThresholdDays: 30
        型：int (0以上の自然数)
        説明：指定した日数が過ぎた場合に、古いスキャンデータであることがわかるようにタブを黄色くします。
        想定値：ユーザーのお好み。

■ 言語ファイルの追加方法
--------------------
本ツールは多言語に対応しており、新しい言語を追加することが可能です。
1. アプリ実行ファイル（.exe）と同じ階層にある「Languages」フォルダを開きます。
2. 既存の「en.yaml」などをコピーし、ファイル名を追加したい言語のカルチャーコード（例：フランス語なら「fr.yaml」）に変更します。
   ※カルチャーコード（例: ja-JP / ja）の一覧は以下を参照してください。
   https://learn.microsoft.com/ja-jp/windows-hardware/manufacture/desktop/available-language-packs-for-windows?view=windows-11
3. YAMLファイル内のテキストを編集します（UTF-8形式で保存してください）。
4. アプリを再起動すると、メニューの「Language」に新しい言語が表示されます。
※必要に応じて、Readme_<language_code>.txtも同様に他のファイルを参考に作成して追加してください。


■ 完全アンインストール（設定・ログの削除）
--------------------
本ツールの設定や実行ログを完全に削除する場合は、以下のフォルダを手動で削除してください。
%LOCALAPPDATA%\Cat & Chocolate Laboratory\LargeFolderFinder
（エクスプローラーのアドレスバーに上記パスを貼り付けると直接開けます）
